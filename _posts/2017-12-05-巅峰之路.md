#### 1.1.2 volatile关键字
有人提点了我一些基础的内容，没错，就是`Java并发`的三大基础之一，`volatile`修饰符。

**`volatile`的基础要点:**
```

```
    注意：并发是java top1问题，volatile是并发的top1问题

#### 1.1.4 线程状态和Monitor
线程状态比较容易理解，但要注意一点：操作系统的线程状态和Java的线程状态是有区别的。
操作系统的线程状态有5个：新建、就绪、运行、阻塞、结束。而Java线程状态则有6种：`new`、`running`、`blocking`、`time wait`、`runnable`、`dead`。
关于Java的线程状态大有学问：
```
1. Java线程的time wait和blocking区别是什么？
2. 线程状态之间的转移条件？
3. 怎么理解线程Monitor
```
Java线程状态转移的内容在`Thread.java`里做了十分详细的注释，建议没看过的看一下，对于打扎实Java多线程编程基础有一定帮助。
线程状态及转移比较基础，面试问到的频率比较低。线程monitor是一个概念，有必要理解一下。

#### 1.1.5 并发集合
基础扎实的同学一定听过`JUC`，没错，并发集合说的就是`java.util.concurrent`的子集（JUC）。


并发集合主要包括几个经典的集合：`ConcurrentHashMap`，`CopyOnWriteArrayList`，`LinkedBlockingQueue`。
`ConcurrentHashMap`是比较常用的数据结构，必须掌握.
`LinkedBlockingQueue`是生产者/消费者模式的实现，也是线程池默认队列，也必须掌握.
`CopyOnWriteArrayList`建议掌握，问得比较少.
```
1.`ConcurrentHashMap` 在JDK1.6采用了分段+数组+链表，其中，段继承了重入锁
2.`ConcurrentHashMap` 在JDK1.7则采用了无锁cas代替了重入锁
3.`ConcurrentHashMap` 在JDK1.8则是将链表部分做了优化，在链表长度大于8时升级为红黑树
4.`LinkedBlockingQueue` 本身就是生产者/消费者模式的实现
```

LinkedBlockingQueue提供了好几个 `生产` 和 `消费` 的方法，它们的区别在于获取`putLock`和`takeLock`时，采用的重入锁的不同`lock`方法：`lock()`和`lockInterupt()
`。详情参考JDK代码，深刻理解这部分代码，对生产者/消费者模型的理解十分有帮助。

上面都是常用/常见的并发/同步集合。除了集合，还有常用的辅助类:`CyclicBarrier`，`CountDownLaunch`。在代码里我只用过`CountDownLaunch`，属于比较常用和必须掌握的基础。

#### 1.1.6 线程池
线程池大家都很熟悉，通常用于处理异步事件。线程池也是面试的高频热点。根据我不同的面试体验，我对线程池这部的内容体会是，每次面试都让我感到我对线程池的认知是不足的。
建议大家读一下线程池的代码，包含几个要点，也是我面试遇到的：

    1.keep_alive参数的作用？
    2.core_size/largest_size含义？
    3.线程池工作流程？
    4.线程的生命周期？

上面是线程池的高频问题，大家可以充分准备一下。另外值得一提的是`线程池的状态`，状态的转移比较难理解，在面试的时候也很少出现。但是理解线程池的状态转移有助于对线程池的理解加深。
我在学习线程池的时候主要是几个方面：
    
    1.worker的实现，生命周期
    2.新创建线程的流程
    3.带返回值的task的.submit(Runnable)方法的实现原理
    
线程池不敢说学得很深，不好在大家面前班门弄斧。多多指教！

#### 1.1.7 Synchronized和ReetrantLock
这个题目基本不用多少，高频题目。它们关键区别在于synchronized是基于字节码实现的，lock是基于信号量实现的。在面试的时候，这两者的尽量讲详细点。
除了实现上，lock还提供了`tryLock()`和`lockInterupt()`方法。这两方法的实现细节见代码详情。

`ReetrantLock`的默认Lock是非公平锁。`Lock`还提供了公平锁。这两者的区别在于获取锁的时候是否进入队列。这是必懂知识点。
而它的实现原理，是在AQS，即`AbstractQueuedSynchronizer`
